# 教育类智能体平台方案（教学通用版）

## Summary
做一个面向教师的通用教学智能体平台，支持教师自定义课程/智能体知识库，后端用 `Java + Spring Boot 3 + MySQL`，智能对话与工作流编排通过 `Dify Workflow` 接入，前端采用 `Ant Design Pro` 作为后台底座，并额外定制未登录官网首页，做成“中文产品首页 + 系统介绍 + 右上角登录 + 登录后按角色进入教师空间或学生空间”的产品形态。  
产品核心是班级互动：教师创建智能体与班级，学生通过班级码加入，在教师可控范围内与智能体互动，教师可查看记录并介入管理。

## Key Changes
- 后端
  - 建立教师、学生、管理员三角色体系。
  - 设计智能体、课程、知识库、班级、会话、消息、任务、反馈等核心表。
  - 封装 `Dify` 工作流调用层，统一处理智能体会话、流式返回、错误重试与记录落库。
  - 知识库策略采用“按智能体/课程绑定”，教师可上传资料并配置为对应智能体的数据源。

- 前端
  - 前端底座优先采用 `Ant Design Pro`（GitHub: https://github.com/ant-design/ant-design-pro），利用其成熟的登录、权限、菜单、后台布局、表格、表单、弹窗和详情页能力，降低首版开发成本。
  - 单独新增未登录首页 `/`，首页面向中国教育与实训场景，展示系统定位、功能亮点、师生互动流程、AI 实训示例、应用场景和登录入口。
  - 保留登录页，登录成功后根据 `token` 中的角色字段进行路由分流：教师进入 `/teacher` 教师空间，学生进入 `/student` 学生空间，管理员进入 `/admin` 管理空间。
  - 教师空间包含智能体管理、课程/实训模块、知识库管理、班级管理、学生互动记录、实训任务与反馈管理。
  - 学生空间包含班级码加入、已加入班级、AI 实训对话、实训任务、提交记录与反馈查看。
  - 菜单和页面权限按角色控制，教师、学生、管理员只看到各自可用的功能入口。

- 互动机制
  - 教师发起或配置智能体，学生通过班级码进入互动空间。
  - 支持教师查看会话、置顶提示、干预回复、布置互动任务。
  - 互动内容以教学问答、知识讲解、作业辅助、课堂延伸为主，保留教学可控性。

## Test Plan
- 验证教师建智能体、绑定知识库、发布班级的全流程。
- 验证学生通过班级码加入、发起对话、收到 Dify 返回结果、会话可追踪。
- 验证不同角色权限隔离，学生不能越权操作教师配置。
- 验证知识库变更后，智能体可正常调用新知识。
- 验证异常场景：Dify 超时、返回空结果、接口失败、知识库未绑定时的降级提示。
- 验证首页、登录页、工作台在桌面端和移动端的基础可用性。

## Assumptions
- `Dify` 只承担工作流编排、知识库与生成能力，业务数据与权限控制放在本系统内。
- 首版不做复杂多租户，只做单校/单组织可演示的教学平台。
- 知识库按“智能体/课程”粒度管理，不做班级级独立知识库。
- 前端优先选 `Ant Design Pro` 做后台底座，原因是其更适合中国教育/政企类后台系统，中文资料多、权限和菜单体系成熟、后续维护和二次开发成本低；未登录首页不直接套后台模板，而是单独定制中文产品展示页。
- 首版重点是“师生互动 + 教师自定义知识库 + 可演示完整闭环”，高级统计、复杂审批、多租户隔离放到后续版本。
