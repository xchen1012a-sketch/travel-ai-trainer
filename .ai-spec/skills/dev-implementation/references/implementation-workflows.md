# 实施工作流

## 新功能

1. 确认验收行为、范围和不做事项。
2. 找到最接近的项目原生实现。
3. 先建立行为测试或可重复验证基线。
4. 按契约、数据、核心逻辑、入口和界面的依赖顺序实施。
5. 每阶段同步权限、日志、文档和验证。

## 缺陷修复

1. 记录现象、环境、输入和预期行为。
2. 稳定复现并定位首个错误状态，不只处理最后一个异常。
3. 用失败测试或可重复脚本固定根因。
4. 实施最小修复，验证相邻边界和回归风险。

## 性能优化

1. 先获得指标、样本、负载和瓶颈证据。
2. 一次只改变一个主要变量。
3. 比较优化前后延迟、吞吐、资源、正确性和成本。
4. 没有测量证据不声明性能提升。

## 重构

1. 明确必须保持不变的外部行为。
2. 先建立特征测试或回归保护。
3. 小步移动职责，避免同时改变行为和结构。
4. 如果范围扩展或暴露业务歧义，暂停并重新确认。

## 数据或配置迁移

1. 明确前向策略、兼容窗口、恢复策略和数据校验。
2. 将结构变更、数据回填和清理拆开。
3. 对大规模变更设计分批、限速、监控和停止条件。
4. “回滚代码”不等于“回滚数据”；分别验证。
