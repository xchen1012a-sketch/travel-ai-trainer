# 集成、权限与配置

## 外部或 AI 集成

外部输入默认不可信。按适用范围实现：身份校验、签名或令牌校验、时间窗口、防重放、字段校验、标准化、幂等、限流、超时、重试、死信、状态查询、回调、日志、审计和人工处置。

不要让外部系统或模型直接绕过业务服务修改核心数据。模型输出默认作为候选结果；自动执行必须具备明确授权、风险阈值、可观测性、停止机制和人工接管。

## 权限

- 在可信执行端强制授权，客户端隐藏不能替代服务端控制。
- 分开考虑功能权限、操作权限、数据范围、字段可见和字段可编辑。
- 默认最小权限，不把个人、部门或组织名称写死在代码中。
- 高风险操作考虑二次确认、审批、双人复核、速率限制和审计。
- 验证越权、横向访问、批量操作、导入导出和失败路径。

## 配置

配置项必须说明类型、默认值、作用域、校验、生效方式、变更权限、审计和回退。机密信息使用项目既有的密钥管理方式，不写入代码、日志、示例或测试快照。

## 闭环验证

验证成功、重复、乱序、超时、部分失败、重试耗尽、人工修正、回调失败和审计查询。仅验证正常路径不算完成。
