# 代码库调查

## 调查顺序

1. 查看项目说明、AI 规范、工作区状态和目录边界。
2. 定位任务入口、相似实现、调用链、依赖和数据所有者。
3. 查找相关测试、契约、配置、权限、迁移、日志和部署定义。
4. 识别生成文件、第三方代码、用户未提交修改和禁止改动区域。
5. 记录可复用模式、拟修改接入点和高风险位置。

## 调查输出

```markdown
## 调查结果
- 已确认事实及证据：
- 相似实现：
- 当前调用链与数据流：
- 权限与信任边界：
- 项目原生命令和约定：
- 推荐接入点：
- 不应修改的位置：
- 假设与待确认事项：
```

## 约束

- 不通过目录名猜测技术栈或模块责任。
- 不读取真实实现就不声称某文件、接口或表存在。
- 搜索范围从任务相关位置逐步扩大，避免无目的加载整个仓库。
- 调查阶段默认只读；用户要求诊断但未要求修复时，到根因和证据为止。
