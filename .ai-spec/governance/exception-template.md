# 规范例外记录

> 每个例外单独保存到项目的 `docs/governance/exceptions/`。

- **例外 ID**：EX-YYYY-NNN
- **规则**：被偏离的文件和条款
- **适用范围**：模块、环境、分支或发布版本
- **业务原因**：为什么当前不能遵守
- **风险**：可能造成的影响
- **补偿控制**：监控、人工复核、额外测试或隔离措施
- **负责人**：具体 Owner
- **批准人**：有权接受风险的人
- **生效时间**：YYYY-MM-DD
- **到期时间**：YYYY-MM-DD
- **关闭条件**：如何恢复遵守规范
- **状态**：proposed / approved / expired / closed

禁止无负责人、无到期时间的永久例外。
