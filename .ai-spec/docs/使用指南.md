# SpecForge 使用指南

## 一句话启动

把本模板放到项目根 `.ai-spec/`，对任意 AI 发送：

```text
请读取 .ai-spec/AI-START.md，并严格按启动协议接入当前项目。先只读识别并输出启动报告，不要直接修改业务代码。
```

这条路径是所有工具的通用回退方式，不依赖专属插件、Skill 或规则格式。

## 安全导入

Windows 可以先执行 dry-run：

```powershell
powershell.exe -NoProfile -ExecutionPolicy Bypass -File .\scripts\install.ps1 `
  -TargetRoot C:\path\to\project `
  -Mode auto `
  -Tools claude-code,codex,cursor
```

确认计划后增加 `-Apply`。默认安装器只创建缺失文件，已有入口和权限配置一律保留并列入冲突清单，不做覆盖或权限并集。

需要完整接入时使用 `-Onboard`：

```powershell
powershell.exe -NoProfile -ExecutionPolicy Bypass -File .\scripts\install.ps1 `
  -TargetRoot C:\path\to\project `
  -Mode auto `
  -Tools claude-code,codex,cursor `
  -Onboard `
  -Apply
```

`-Onboard` 会执行多项目识别、生成父级 `.specforge.json`、为子项目独立安装 `.ai-spec/`，并删除已安装规范中对当前项目无关的模板文件。它还会生成裁剪版 `business-rules.md`：未检测到 KPI、组织身份、状态机、权限菜单或外部集成信号时，不保留对应空章节。

日常会话不应从完整 `AI-START.md` 开始。接入完成且 `quick-ref.md` 状态为 `GENERATED` 后，AI 应先读 `business/quick-ref.md`，再按其中的动态上下文门禁按需加载 `core-lite/`、`business/`、`contracts/` 或完整 `core/`。安装器会根据文件数量、构建文件、API、DB、Auth、CI、多项目等信号写入 `projectSize`（tiny / small / medium / large / enterprise），不同规模使用不同默认加载策略。Git 分支和无 Git 仓库的初始基线 commit 需要额外显式传入 `-ManageGit`。

多项目接入会在父目录 `.specforge.json` 和各子项目 `.ai-spec/ai-spec.yaml` 记录 `templateVersion`。如果后续模板升级导致版本不一致，先 dry-run 查看同步计划：

```powershell
powershell.exe -NoProfile -ExecutionPolicy Bypass -File .\scripts\install.ps1 `
  -TargetRoot C:\path\to\parent-or-project `
  -Sync
```

确认后增加 `-Apply`。`-Sync` 只更新 `AI-START.md`、`README.md`、`core/`、`core-lite/`、`contracts/`、`governance/`、`skills/`、`workflows/` 和 `scripts/validate.ps1`，不会覆盖 `ai-spec.yaml`、`business/`、`stacks/`、`adapters/` 等项目差异文件。

没有 PowerShell 时，直接复制本目录到目标项目 `.ai-spec/`，再使用上面的通用启动提示词即可。

## 新项目

推荐使用自动接入模式（`AI-START.md` §1.5），AI 会自动建分支、同步规范、扫描业务、生成报告。

手动接入（备选）：
1. 复制模板到 `.ai-spec/`。
2. 复制 `ai-spec.example.yaml` 为 `ai-spec.yaml`。
3. 让 AI 执行启动协议并生成项目画像草案。
4. 确认画像后执行 `workflows/new-project.md`。
5. 按使用工具安装 `adapters/` 中的入口。
6. 运行 `scripts/validate.ps1`。

## 老项目

推荐使用自动接入模式（`AI-START.md` §1.5），AI 会建分支保护、同步规范、扫描项目生成业务规则。

手动接入（备选）：
1. 不覆盖已有 `CLAUDE.md`、`AGENTS.md`、Cursor 规则或权限配置。
2. 让 AI 按 `workflows/existing-project.md` 做只读体检和差异报告。
3. 确认 dry-run 后分阶段接入。
4. 新代码先遵守，旧代码按模块逐步治理。

## 开发中项目

推荐使用自动接入模式（`AI-START.md` §1.5），按 in-progress 阶段处理。

手动接入（备选）：
1. 先记录当前分支、未提交改动、契约和未完成事项。
2. 只启用启动入口、安全底线和交付协议。
3. 当前功能完成后再做全量规范迁移。
4. 不自动 stash、格式化或移动当前业务文件。

## 工具入口

专用入口见 `adapters/README.md`。目标文件存在时必须合并，不得覆盖。未知工具直接使用 `adapters/generic/START-PROMPT.md`。
