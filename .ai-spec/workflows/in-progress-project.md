# 开发中项目接入工作流

> **注意**：AI-START.md §1.5 自动接入模式已处理大部分本文件描述的步骤。以下仅在非自动模式或用户明确要求时适用。

适用于存在未完成需求、功能分支、未提交改动、联调或迁移中的项目。

## 最高优先级

保护当前开发状态，避免接入规范改变正在实施的契约和业务语义。

## 流程

1. 记录 Git 状态、当前分支、未提交文件、进行中任务和已有 Handoff。
2. 明确本轮接入是否允许修改当前功能涉及的文件。
3. 只接入 `AI-START.md`、安全底线和交付协议，先不改变架构和分支模型。
4. 对当前需求建立“现行约定快照”：接口、数据库、权限、测试和未验证项。
5. 当前阶段结束后再执行完整差异评估和模块化迁移。
6. 若当前改动已偏离规范，先报告，不在未授权情况下顺手修正。

## 禁止

- 为接入规范清理或 stash 用户工作区。
- 在当前功能中混入全项目格式化、目录搬迁或依赖升级。
- 把未完成状态写成已验证事实。

## 输出

当前状态快照、最小接入范围、冲突清单、当前阶段完成条件和后续迁移计划。
