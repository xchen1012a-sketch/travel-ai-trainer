# 项目命令规范

> AI 不猜命令。项目的真实命令来自构建清单、脚本和 `ai-spec.yaml`。

## 一、标准命令能力

项目按适用范围提供下列逻辑能力，具体命令由技术栈决定：

| 能力 | 用途 | 要求 |
|---|---|---|
| setup | 安装或恢复依赖 | 使用锁文件，可重复执行 |
| format | 格式化代码 | 范围可控，不顺带格式化全仓 |
| lint | 静态规则检查 | CI 与本地一致 |
| typecheck | 类型检查 | 不用跳过或宽松参数掩盖错误 |
| build | 生成可发布制品 | 非交互，失败码准确 |
| test | 默认测试入口 | 明确包含哪些测试层级 |
| test:unit | 单元测试 | 快速、隔离 |
| test:integration | 集成测试 | 声明依赖和数据隔离方式 |
| test:e2e | 端到端测试 | 声明环境、账号和清理策略 |
| run/dev | 本地运行 | 端口、依赖和停止方式明确 |
| migrate | 数据库迁移 | 环境显式、支持 dry-run/状态检查 |
| validate | 项目统一质量检查 | 组合适用的 lint/typecheck/test/build |

不适用的能力可以省略，但必须在项目画像中明确，而不是填入空命令冒充支持。

## 二、AI 执行规则

- 优先使用项目已有脚本，不绕过包管理器或构建工具直接调用内部实现。
- 执行前说明会产生的文件、网络、容器、数据库或外部服务副作用。
- 默认使用非交互和确定性参数；禁止依赖等待人工输入的隐藏步骤。
- 先运行受影响范围的快速检查，再根据风险运行更广回归。
- 不使用 `|| true`、吞错误或伪造退出码让流水线变绿。
- 不自动执行 deploy、publish、release、生产 migrate、批量通知和付费资源命令。
- 命令失败时保留关键错误、退出码和复现方式，不只报告“执行失败”。

## 三、多仓和 Monorepo

- 每条命令记录工作目录，禁止在协调父目录误执行仓库写操作。
- Monorepo 优先使用受影响模块能力，但发布候选按项目策略运行必要的全局门禁。
- 多仓任务分别记录各仓命令和结果，不把一仓通过当作全栈验证完成。

## 四、命令变更

新增或修改标准命令时同步更新：

1. 构建清单或任务脚本；
2. `ai-spec.yaml`；
3. CI/CD；
4. 开发文档；
5. 对应验证测试。
