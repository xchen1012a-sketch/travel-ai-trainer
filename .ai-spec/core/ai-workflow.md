# 多 AI 协作工作流

> Claude / Codex / Cursor / 任何 AI 编码助手都是**可替换的执行器**。
> 真正的状态源是：**Git（代码）+ `.ai-spec/`（规则）+ 契约文档 + Handoff**。
> AI 对话记忆**不是**状态源。

---

## 一、工具定位（不绑定具体工具）

| 工具类型 | 擅长 | 不擅长 |
|---|---|---|
| **方案/推理能力强** | 架构设计、复杂业务拆解、契约初稿、产品方案 | 成本和延迟可能较高 |
| **实施/工具能力强** | 代码实现、Bug 修复、局部优化、验证、文档 | 仍需明确需求和风险边界 |
| **IDE 内嵌型** | 边写边补全、局部重构 | 自动发现和跨仓上下文能力因工具而异 |

**原则**：按任务难度选工具，不要"用一把锤子敲所有钉子"。

---

## 二、双 Skill 模式（推荐）

项目共用两个标准 Skills：`../skills/product-architect/SKILL.md` 和 `../skills/dev-implementation/SKILL.md`：

```
业务想法 / 需求
   ↓
Skill A：产品架构师    → 输出：产品方案 + 架构 + 数据模型 + 接口契约 + 权限码
   ↓
用户确认方案
   ↓
Skill B：开发实施      → 输出：规划调研 → 实施计划 → 小步开发 → 验证交付
```

**为什么分两个 Skill**：
- 方案和实施关注点不同（一个想清楚，一个做出来）
- 中间有**用户确认环节**，防止 AI 一路狂奔偏离需求
- 不同 AI 工具都能套用这套模式

---

## 三、状态源（State Source）

| 状态 | 存哪里 | 备注 |
|---|---|---|
| 代码 | Git | 唯一权威 |
| 规则 | `.ai-spec/` | 唯一权威 |
| 接口契约 | `docs/contracts/{feature}.md` | 跨端协作时 |
| 任务交接 | 项目的 `docs/handoffs/` | 结构参考 `governance/handoff-template.md` |
| AI 对话记忆 | ❌ 不是状态源 | 会话结束即消失 |

---

## 四、规划阶段（复杂任务必走）

复杂任务（跨模块、跨契约、架构/数据/权限变更或高风险）必须先完成规划阶段；工具没有专用 Plan Mode 时，用普通对话输出同等计划：

```
1. AI 读相关代码
2. AI 输出实施计划：
   - 修改哪些文件
   - 每个文件改什么
   - 依赖顺序
   - 风险点
   - 验证方式
3. 存在重大业务或架构决策时由用户/Owner 确认；目标和方案已明确时可继续实施
4. AI 按计划小步实施
5. 每步给验证 + 风险 + 回滚
```

**禁止**：复杂任务在未调查影响面、风险和验证方式时直接大改。

---

## 五、Handoff（任务交接）

切换工具 / 切换会话 / 切换人时，必须生成 Handoff。

### 5.1 Handoff 模板

使用 `../governance/handoff-template.md`。Handoff 必须记录目标、工作区状态、已完成/未完成、修改和契约、验证证据、风险、恢复方式及下一步；禁止包含密码、Token 和个人敏感数据。

### 5.2 接手后第一步

1. 读 `.ai-spec/AI-START.md`（规范位于项目根时读 `AI-START.md`）
2. 读当前任务 Handoff
3. `git status --short` + `git log --oneline -10`
4. 读 Handoff 中列出的文件和契约
5. 确认当前阶段目标，再继续

---

## 六、AI 输出约束

任何 AI 在本项目中输出代码 / 方案时必须：

- **先读后改**（不知道现状就别动手）
- **不顺手扩大范围**（用户问 A，AI 不能顺便改 B）
- **每次都给四件套**（验证 / 未验证 / 风险 / 回滚）
- **不夸大完成度**（"已完成"必须真的验证过）
- **不隐瞒风险**（高风险操作必须事前说）

---

## 七、常见反模式

| 反模式 | 表现 | 后果 |
|---|---|---|
| **不读就改** | "看起来这里应该改一下" | 改乱现有逻辑 |
| **过度发挥** | 用户问 A，AI 改了 A+B+C+D | 范围爆炸，无法 review |
| **空安慰** | "这个改完应该没问题" | 未验证就交付 |
| **隐藏风险** | 改了认证逻辑但不告知 | 上线后炸 |
| **依赖记忆** | "上次我们说..."（但没文档化） | 切换工具就丢 |
| **跳过规划** | 复杂任务直接写码 | 返工 + 偏离需求 |

---

## 八、工具切换检查清单

切换 AI 工具 / 会话前：

- [ ] Handoff 已按治理模板填写
- [ ] Git/工作区状态已准确记录；只有用户要求或团队流程要求时才提交
- [ ] 当前分支名已记录
- [ ] 接口契约已更新到 `docs/contracts/`
- [ ] 未解决问题已列在 Handoff 的"下一步指令"
- [ ] 涉及高风险的，已显式标注（删数据 / 改权限 / 改业务口径）
